## 3. subway

路径：`/nas/lyh/Awesome_LT/REBOUND/data_sub/subway`

### README 中与数据有关的介绍

- 数据主题：`上海地铁一天时间内的人流量变化`

### 数据相关文件

#### `data/stations_with_geo.csv`

- 行数：`303`
- 列数：`5`
- 无标准表头

样例：

- `共康路, 36.5265648, 135.2028491, 1, 0`

推测字段：

| 列位置 | 含义 |
| --- | --- |
| 第 1 列 | 站点名 |
| 第 2 列 | 纬度或投影坐标 |
| 第 3 列 | 经度或投影坐标 |
| 第 4 列 | 线路编号 |
| 第 5 列 | 站点类别/标记位；在可视化代码里该值为 `1` 时会画成更大的点，可能表示换乘站或重点站 |

#### `data/links_with_geo.csv`

- 行数：`350`
- 列数：`10`
- 无标准表头

样例：

- `市光路, 36.5306638, 135.2975413, 嫩江路, 36.5208181, 135.2975766, 8, 0, 158, 218`

字段含义：

| 列位置 | 含义 |
| --- | --- |
| 1 | 起点站名 |
| 2 | 起点纬度 |
| 3 | 起点经度 |
| 4 | 终点站名 |
| 5 | 终点纬度 |
| 6 | 终点经度 |
| 7 | 线路编号 |
| 8 | 颜色参数 `R` |
| 9 | 颜色参数 `G` |
| 10 | 颜色参数 `B` |

说明：

- 这份文件的核心作用不是记录客流，而是记录“地铁线路图上的连边”
- 在 [subway.pde](/nas/lyh/Awesome_LT/REBOUND/data_sub/subway/subway.pde) 里，每一行会被画成一条站点之间的连线
- 最后 3 列会直接作为 `RGB` 颜色值使用，用来区分不同线路的显示颜色

#### `data/oneday_traffic.csv`

- 行数：`2879`
- 列数：`2625`
- 无标准表头

说明：

- 这不是普通二维表，而是给 [subway.pde](/nas/lyh/Awesome_LT/REBOUND/data_sub/subway/subway.pde) 直接使用的可视化输入数据
- 它大致可以分成两段：

第一段：前 `1440` 行

- 对应一天 `1440` 分钟的逐分钟可视化帧
- 每一行包含大量重复的 7 列一组数据
- 每组 7 列可理解为：
  - `纬度`
  - `经度`
  - `R`
  - `G`
  - `B`
  - `进站流量`
  - `出站流量`

也就是说，这一段本质上是在描述：

- 某一分钟里，每个站点画在什么位置
- 用什么颜色画
- 该时刻进站/出站圆圈要画多大

第二段：后 `1440` 行

- 不是站点逐帧数据，而是“排名信息”
- 每行看起来是“站点名 + 数值”的成对数据
- 在 `subway.pde` 里，这一段被用来显示某个 10 分钟窗口内的站点客流排行
- 代码按如下方式读取：
  - 前 20 列：10 组“站点名 + 数值”
  - 后 20 列：另一组 10 个“站点名 + 数值”
- 可理解为：
  - 一组是进站排行
  - 一组是出站排行

因此，这个文件的核心意义可以总结为：

- 前半段：逐分钟站点流量可视化底层数据
- 后半段：按时间片汇总的站点排行数据

样例理解：

- 第 0 行开头是：
  - `36.5265648,135.2028491,233,27,57,0.00,0.00`
  - 含义像是：
    - 一个站点的位置
    - 该站点的显示颜色 `RGB(233,27,57)`
    - 当前分钟进站 `0.00`
    - 当前分钟出站 `0.00`
- 第 1440 行开头是：
  - `共康路,0,马陆,0,淞虹路,0,...`
  - 这已经不是坐标流量，而是站点名和对应排名值

## 4. 按文件逐个总结

| 文件 | 所属数据集 | 文件内容总结 |
| --- | --- | --- |
| `MetroFlow/Data/stationInfo.csv` | `MetroFlow` | 302 个上海地铁站点的基础信息表，包含站点 ID、名称、经纬度和邻接站点列表。 |
| `metro_prediction/data/*.csv` | `metro_prediction` | 按站点拆分的真实客流时间序列。每个文件对应一个站点，字段为“时 / 第x个五分钟 / 客流量”。 |
| `metro_prediction/KNN_output/*.csv` | `metro_prediction` | 按站点拆分的 KNN 预测结果时间序列，结构与 `data/*.csv` 一致。 |
| `metro_prediction/LSTM_output/*.csv` | `metro_prediction` | 按站点拆分的 LSTM 预测结果时间序列，包含真实客流量和预测客流量两列。 |
| `metro_prediction/station_info_json.json` | `metro_prediction` | 站点信息 JSON，用于地图展示或站点层可视化。 |
| `subway/data/stations_with_geo.csv` | `subway` | 上海地铁站点位置表，记录站点名、坐标、线路编号，以及一个用于绘图区分的站点类别标记。 |
| `subway/data/links_with_geo.csv` | `subway` | 上海地铁线路连边表，记录相邻站点之间的连接关系，以及画线路时使用的 RGB 颜色。 |
| `subway/data/oneday_traffic.csv` | `subway` | 一天内地铁流量的可视化底层数据文件。前半段是逐分钟站点流量帧，后半段是站点排行信息。 |
