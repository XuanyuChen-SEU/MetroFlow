# MetroFlow 项目结构设计

本文档定义 MetroFlow 的目录组织、职责边界与落盘约束，作为前端、后端、脚本与文档协同开发的结构基准。

## 1. 顶层结构

```text
final-project/
├── backend/
├── data/
├── docs/
├── frontend/
├── scripts/
├── sql/
├── demo.html
└── README.md
```

## 2. 目录职责

- `frontend/`：Vue 3 前端工程
- `backend/`：Flask 接口服务
- `scripts/`：数据清洗、转换与导出脚本
- `data/`：原始数据与处理后数据
- `sql/`：数据库相关脚本
- `docs/`：项目文档
- `demo.html`：单文件联调页面

## 3. 前端结构

```text
frontend/
├── index.html
├── package.json
├── vite.config.js
└── src/
    ├── api/
    ├── assets/
    ├── components/
    ├── utils/
    ├── views/
    ├── App.vue
    └── main.js
```

### 3.1 前端分层原则

- `views/`：页面级容器与布局
- `components/`：可复用组件
- `api/`：HTTP 请求封装
- `utils/`：图表配置、地理转换、辅助函数
- `assets/`：样式、静态 JSON 与图形资源

## 4. 后端结构

```text
backend/
├── app.py
├── config.py
├── requirements.txt
├── api/
├── db/
├── services/
└── utils/
```

### 4.1 后端职责划分

- `app.py`：应用入口与蓝图注册
- `config.py`：运行配置
- `api/`：HTTP 路由层
- `services/`：业务聚合与数据读取逻辑
- `db/`：数据库连接封装
- `utils/`：统一响应与辅助函数

## 5. 数据与脚本结构

```text
data/
├── count_df.csv
├── metro_global.json
├── station_info.json
└── processed/
    ├── dim_station.csv
    ├── fact_hourly_flow.csv
    ├── macro_city_flow_clean.csv
    └── shanghai_line_topology.json

scripts/
├── clean_data.py
├── export_seed_csv.py
└── load_mysql.py
```

### 5.1 数据目录原则

- `data/` 根目录仅存放原始输入数据
- `data/processed/` 存放标准化中间产物
- 不在业务代码目录中存放原始数据副本

### 5.2 脚本职责

- `clean_data.py`：生成标准化中间数据
- `export_seed_csv.py`：输出汇总检查信息
- `load_mysql.py`：导入 MySQL（可选模式）

## 6. 文档目录

```text
docs/
├── 快速启动.md
├── 使用指南.md
├── 项目结构设计.md
└── 系统设计指南.md
```

## 7. 结构约束

1. 前端代码仅写入 `frontend/`
2. 后端代码仅写入 `backend/`
3. 数据脚本仅写入 `scripts/`
4. SQL 文件仅写入 `sql/`
5. 原始数据文件不得移动或重命名
6. 新增中间产物统一输出到 `data/processed/`

## 8. 命名规则

- Vue 页面组件：大驼峰命名，如 `ShanghaiDetail.vue`
- JavaScript 模块：语义化小写命名
- Python 文件：下划线命名，如 `clean_data.py`
- SQL 文件：语义化命名，如 `schema.sql`
