# MetroFlow 使用指南

## 1. 你现在拿到的内容

当前仓库已经补齐了 4 类内容：

- `backend/` Flask 接口服务
- `frontend/` Vue3 + Vite 前端骨架
- `demo.html` 单文件联调页面
- `scripts/` 数据清洗与 MySQL 导入脚本

## 2. 推荐启动顺序

建议按下面顺序操作：

1. 先生成 `processed CSV`
2. 启动 Flask 后端
3. 用 `demo.html` 先测接口和交互
4. 再启动 Vue3 前端工程
5. 如果需要数据库，再执行 MySQL 建表和导入

## 3. 第一步：生成清洗后的数据

在项目根目录执行：

```bash
python scripts/clean_data.py
```

执行成功后会生成：

- `data/processed/dim_station.csv`
- `data/processed/fact_hourly_flow.csv`
- `data/processed/macro_city_flow_clean.csv`

这一步必须先做。因为后端默认从 `data/processed/` 读取数据。

## 4. 第二步：启动后端

进入后端目录：

```bash
cd backend
pip install -r requirements.txt
python app.py
```

默认地址：

- `http://127.0.0.1:5000`

启动后可直接测试：

- `http://127.0.0.1:5000/`
- `http://127.0.0.1:5000/api/overview`
- `http://127.0.0.1:5000/api/overview/cities`
- `http://127.0.0.1:5000/api/shanghai/flow?hour=8`
- `http://127.0.0.1:5000/api/shanghai/lines?hour=8`
- `http://127.0.0.1:5000/api/shanghai/station/人民广场`

## 5. 第三步：如何加载数据

### 5.1 默认方式：CSV 直读

后端默认配置：

- `DATA_SOURCE=csv`

这意味着 Flask 会直接读取 `data/processed/*.csv`，不依赖 MySQL。适合你先开发和联调后端接口。

### 5.2 切换到 MySQL

先在 MySQL 中创建库并执行：

```bash
mysql -u root -p < sql/schema.sql
```

然后回到项目根目录执行：

```bash
python scripts/load_mysql.py
```

再设置环境变量后启动后端：

```bash
set DATA_SOURCE=mysql
set MYSQL_HOST=127.0.0.1
set MYSQL_PORT=3306
set MYSQL_USER=root
set MYSQL_PASSWORD=123456
set MYSQL_DATABASE=metroflow
python backend/app.py
```

如果你使用 PowerShell，也可以写成：

```powershell
$env:DATA_SOURCE="mysql"
$env:MYSQL_HOST="127.0.0.1"
$env:MYSQL_PORT="3306"
$env:MYSQL_USER="root"
$env:MYSQL_PASSWORD="123456"
$env:MYSQL_DATABASE="metroflow"
python backend/app.py
```

## 6. 第四步：使用 `demo.html` 单文件测试

`demo.html` 的用途是：

- 不依赖 Vue 工程
- 只要后端起来，就能直接测页面和接口
- 方便你和前端同学并行开发

使用方式：

1. 先确保 Flask 已经启动
2. 直接用浏览器打开根目录下的 `demo.html`
3. 页面会请求 `http://127.0.0.1:5000/api/...`

如果浏览器限制本地文件跨域，建议用一个静态服务器打开：

```bash
python -m http.server 8080
```

然后访问：

- `http://127.0.0.1:8080/demo.html`

## 7. 第五步：启动 Vue3 前端

进入前端目录：

```bash
cd frontend
npm install
npm run dev
```

默认地址：

- `http://127.0.0.1:5173`

前端 `vite.config.js` 已经把 `/api` 代理到：

- `http://127.0.0.1:5000`

所以只要后端启动，Vue 页面就能直接调接口。

## 8. 当前后端接口说明

目前提供 5 个主要接口：

1. `GET /api/overview`
   - 返回指标卡、Top10、趋势图数据
2. `GET /api/overview/cities`
   - 返回全国城市气泡图数据
3. `GET /api/shanghai/flow?hour=8`
   - 返回指定小时上海站点散点数据
4. `GET /api/shanghai/lines?hour=8`
   - 返回指定小时线路热力条数据
5. `GET /api/shanghai/station/人民广场`
   - 返回单站点 24 小时曲线数据

## 9. 你和前端同学怎么分工

推荐分工方式：

- 你负责 `backend/`、`sql/`、`scripts/`
- 前端同学负责 `frontend/src/components/`、`frontend/src/views/`
- 双方先用 `demo.html` 对接口结构达成一致

## 10. 常见问题

### 10.1 为什么后端启动后没有数据

通常是因为没先运行：

```bash
python scripts/clean_data.py
```

### 10.2 为什么 `demo.html` 打不开图表

优先检查：

- Flask 是否启动
- 浏览器控制台是否报跨域
- 是否通过 `python -m http.server 8080` 提供静态访问

### 10.3 为什么 MySQL 模式报错

优先检查：

- 是否执行了 `sql/schema.sql`
- 是否运行了 `python scripts/load_mysql.py`
- 环境变量是否填写正确

## 11. 建议的开发顺序

建议你现在这样推进：

1. 先跑 `python scripts/clean_data.py`
2. 再跑 `python backend/app.py`
3. 用 `demo.html` 把 5 个接口全测通
4. 前端同学再基于 `frontend/` 接着细化 Vue 页面
