# MetroFlow 使用指南

本文档说明 MetroFlow 在本地环境中的标准使用方式，包括数据生成、接口联调与前端运行。

## 1. 运行模式

系统默认采用 CSV 直读模式：

- `DATA_SOURCE=csv`

在该模式下，后端直接读取 `data/processed/` 下的处理后文件，不依赖 MySQL。

## 2. 数据处理

### 2.1 输入数据

- `data/count_df.csv`
- `data/metro_global.json`
- `data/station_info.json`

### 2.2 处理命令

```bash
python scripts/clean_data.py
```

### 2.3 输出文件

- `data/processed/dim_station.csv`
- `data/processed/fact_hourly_flow.csv`
- `data/processed/macro_city_flow_clean.csv`
- `data/processed/shanghai_line_topology.json`

## 3. 后端运行

```bash
cd backend
pip install -r requirements.txt
python app.py
```

默认服务地址：

- `http://127.0.0.1:5000`

### 3.1 常用接口

- `GET /api/overview`
- `GET /api/overview/cities`
- `GET /api/shanghai/topology`
- `GET /api/shanghai/flow?hour=8`
- `GET /api/shanghai/lines?hour=8`
- `GET /api/shanghai/load-trend?line=all`
- `GET /api/shanghai/station/人民广场`

## 4. 前端运行

```bash
cd frontend
npm install
npm run dev
```

默认访问地址：

- `http://127.0.0.1:5173`

## 5. demo.html 联调

`demo.html` 用于后端接口快速验证和单文件页面调试。

使用方式：

1. 启动 Flask 后端
2. 直接打开 `demo.html`

如需避免本地文件协议限制，可执行：

```bash
python -m http.server 8080
```

然后访问：

- `http://127.0.0.1:8080/demo.html`

## 6. 常见问题

### 6.1 接口无数据

通常原因如下：

- 未执行 `python scripts/clean_data.py`
- `data/processed/` 文件缺失
- 后端未正确启动

### 6.2 上海地图未显示线路

优先检查：

- `data/processed/shanghai_line_topology.json` 是否存在
- 是否重新执行过 `scripts/clean_data.py`

### 6.3 前端页面无法请求接口

优先检查：

- Flask 是否运行在 `127.0.0.1:5000`
- Vite 代理配置是否生效
- 浏览器控制台是否存在跨域或网络错误
