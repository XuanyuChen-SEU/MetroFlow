
# MetroFlow（轨交脉搏）系统设计指南

> 面向：东南大学 虚拟现实与数据可视化课程项目
> 技术栈：Vue3 + ECharts + Flask + MySQ

---

## 1. 系统概述

### 1.1 项目名称

MetroFlow（轨交脉搏）——上海地铁客流时空可视化系统

### 1.2 项目目标

- 构建全国地铁宏观客流可视化大屏
- 构建上海地铁微观小时级拥挤度时空可视化大屏
- 支持时间滑块、动态回放、站点详情、线路负荷热力条
- 深色大屏风格，可用于运营、商业选址、公众出行参考

### 1.3 核心能力

- 全国 54 城地铁客流气泡地图、排名、趋势
- 上海地铁全网站点散点拓扑图（位置、颜色、大小）
- 24 小时时间轴交互、自动回放、单站点全天曲线
- 线路实时拥挤度热力条、换乘站视觉突出

---

## 2. 总体架构设计

### 2.1 架构分层（四层）

1. **前端展示层**：Vue3 + ECharts（Tab1 全国、Tab2 上海）
2. **交互控制层**：Vue 响应式、watch、v-model、事件绑定
3. **后端服务层**：Flask RESTful API、跨域支持
4. **数据存储层**：MySQL + JSON 静态数据

### 2.2 数据流

```
原始数据（CSV） → Python 清洗 → MySQL 入库
前端 → Axios → Flask API → MySQL → JSON → ECharts 渲染
滑块/回放 → Vue watch → 请求新数据 → 图表重绘
```

### 2.3 技术栈选型

- 前端：Vue3（Vite）、ECharts 5.x、Axios、原生 CSS（深色）
- 后端：Python 3.9+、Flask、Flask-CORS
- 数据库：MySQL 8.0
- 数据处理：Pandas、NumPy、坐标转换

## 3. 数据库设计

### 3.1 表结构（共 3 张）

#### 3.1.1 dim_station（上海站点静态表）

```sql
CREATE TABLE dim_station (
  station_id INT PRIMARY KEY AUTO_INCREMENT,
  station_name VARCHAR(50) NOT NULL,
  line_name VARCHAR(50) NOT NULL,
  lat DOUBLE NOT NULL,
  lng DOUBLE NOT NULL,
  is_transfer TINYINT DEFAULT 0 COMMENT '1=换乘站'
);
```

#### 3.1.2 fact_hourly_flow（小时拥挤度动态表）

```sql
CREATE TABLE fact_hourly_flow (
  id INT PRIMARY KEY AUTO_INCREMENT,
  time_hour VARCHAR(20) NOT NULL COMMENT '00:00~23:00',
  station_id INT NOT NULL,
  crowd_index INT NOT NULL COMMENT '0-100',
  FOREIGN KEY (station_id) REFERENCES dim_station(station_id),
  INDEX idx_time (time_hour)
);
```

#### 3.1.3 macro_city_flow（全国城市宏观表）

```sql
CREATE TABLE macro_city_flow (
  city_id INT PRIMARY KEY AUTO_INCREMENT,
  city_name VARCHAR(30) NOT NULL,
  avg_flow DOUBLE NOT NULL COMMENT '日均客流（万人次）',
  line_count INT,
  total_length DOUBLE,
  lat DOUBLE,
  lng DOUBLE
);
```

### 3.2 静态数据文件

- `macro_map_data.json`：全国城市（54城）名称、经纬度、日均客流，前端直接加载，不查库

## 4. 数据设计

### 4.1 数据源

- 宏观：交通运输部公开数据（2025年全国54城）
- 微观：上海地铁站点拓扑（1/2/3/4/10号线为主）+ 模拟小时客流

### 4.2 数据字段映射

- 地理坐标 → 地图定位
- 客流数值 → 气泡大小、柱状高度
- 拥挤度（0-100）→ 散点颜色（绿/黄/红）
- 线路名称 → 筛选、热力条分组
- 时间切片 → 滑块、回放、曲线横轴

### 4.3 数据预处理流程

1. 坐标统一：GCJ-02 → WGS84
2. 异常值清洗：去空、去负
3. 归一化：客流 → 0-100 拥挤度
4. 入库：dim_station、fact_hourly_flow、macro_city_flow
5. 导出：macro_map_data.json

## 5. 后端 API 设计

### 5.1 接口清单（共5个）

1. `GET /api/macro`

   - 功能：获取全国城市客流数据
   - 返回：`[{city_name, avg_flow, lat, lng, line_count, total_length}]`
2. `GET /api/flow?time=HH:00`

   - 功能：获取上海某小时所有站点拥挤度
   - 返回：`[{station_id, station_name, lat, lng, crowd_index, line_name}]`
3. `GET /api/station?sid=xxx`

   - 功能：获取单个站点全天24小时拥挤度
   - 返回：`[{time_hour, crowd_index}]`
4. `GET /api/lineheat`

   - 功能：获取上海各线路平均拥挤度
   - 返回：`[{line_name, avg_crowd}]`
5. `GET /api/top10`

   - 功能：全国客流 Top10
   - 返回：`[{city_name, avg_flow}]`

### 5.2 技术要求

- 跨域：启用 CORS
- 格式：统一 JSON
- 性能：时间字段加索引，支持毫秒级查询

### 6.2 组件设计

- `TrendLine.vue`：趋势折线
