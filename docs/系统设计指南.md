# MetroFlow 系统设计指南

本文档说明 MetroFlow 的系统目标、数据组织、接口设计与页面设计。

## 1. 系统目标

MetroFlow 用于展示地铁客流在宏观城市层面与微观站点层面的时空分布特征，面向课程项目演示、本地联调与可视化分析场景。

系统由两个核心页面组成：

- 全国概览：展示全国城市客流分布、排名与趋势
- 上海微观：展示上海地铁线路拓扑、站点客流与小时级变化

## 2. 总体架构

系统采用前后端分离架构。

### 2.1 展示层

- Vue 3
- ECharts 5
- Axios

### 2.2 服务层

- Flask
- 基于 REST 风格的 JSON 接口

### 2.3 数据层

- 原始数据：CSV 与 JSON
- 处理中间层：`data/processed/`
- 可选存储层：MySQL

## 3. 数据设计

## 3.1 原始数据

- `count_df.csv`：上海站点小时客流基础数据
- `metro_global.json`：全国城市客流数据
- `station_info.json`：上海地铁线路与站点拓扑信息

## 3.2 处理后数据

### 3.2.1 `dim_station.csv`

站点维表，保存站点名称、线路集合、换乘标记与拓扑坐标。

### 3.2.2 `fact_hourly_flow.csv`

小时客流事实表，保存每个站点在 `0-23` 小时维度上的平均客流与拥挤指数。

### 3.2.3 `macro_city_flow_clean.csv`

全国城市宏观表，保存城市在最近时间窗口内的客流快照。

### 3.2.4 `shanghai_line_topology.json`

上海线路拓扑文件，保存线路颜色、坐标序列与全图范围。

## 4. 接口设计

### 4.1 全国概览

- `GET /api/overview`
- `GET /api/overview/cities`

### 4.2 上海微观

- `GET /api/shanghai/topology`
- `GET /api/shanghai/flow?hour=8`
- `GET /api/shanghai/lines?hour=8`
- `GET /api/shanghai/load-trend?line=all`
- `GET /api/shanghai/station/<station>`

## 5. 页面设计

## 5.1 全国概览

页面包含以下模块：

- 指标卡片
- 全国客流分布图
- 客流 Top10 城市
- 趋势对比
- 摘要信息

## 5.2 上海微观

页面包含以下模块：

- 小时级时间轴与播放控制
- 上海线路拓扑图
- 行政区聚焦
- 线路筛选
- 线路热力条
- 全天负荷趋势
- 热点站点 Top5

## 6. 可视编码原则

- 全国页：气泡大小表示客流规模
- 上海页：站点大小表示小时客流，颜色表示拥挤等级
- 线路颜色采用固定色板，保持可辨识性
- 拥挤等级分为无客流、畅通、繁忙、拥挤四档

## 7. 运行模式

### 7.1 默认模式

- `DATA_SOURCE=csv`

后端直接读取 `data/processed/` 下的文件。

### 7.2 可选模式

- `DATA_SOURCE=mysql`

该模式下后端从 MySQL 读取对应业务表。当前项目运行与联调优先推荐 CSV 模式。
