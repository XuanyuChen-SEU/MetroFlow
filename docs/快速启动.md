# MetroFlow 快速启动

这份文档只保留当前最短可运行路径：

- 使用 `CSV` 直读模式
- 不涉及 `MySQL`
- 适合本地联调和演示

## 1. 前置说明

项目当前使用的核心数据源：

- `data/count_df.csv`
- `data/metro_global.json`
- `data/station_info.json`

运行前需要先生成处理后数据，后端和前端都会依赖：

- `data/processed/dim_station.csv`
- `data/processed/fact_hourly_flow.csv`
- `data/processed/macro_city_flow_clean.csv`
- `data/processed/shanghai_line_topology.json`

## 2. 第一步：生成处理后数据

在项目根目录执行：

```bash
python scripts/clean_data.py
```

如果执行成功，终端会看到类似输出：

```text
dim_station rows: 344
fact_hourly_flow rows: 8256
macro_city_flow rows: 540
unmatched flow stations: 0
```

## 3. 第二步：启动 Flask 后端

进入后端目录：

```bash
cd backend
pip install -r requirements.txt
python app.py
```

默认启动地址：

- `http://127.0.0.1:5000`

可以直接检查这些接口是否可访问：

- `http://127.0.0.1:5000/`
- `http://127.0.0.1:5000/api/overview`
- `http://127.0.0.1:5000/api/overview/cities`
- `http://127.0.0.1:5000/api/shanghai/topology`
- `http://127.0.0.1:5000/api/shanghai/flow?hour=8`
- `http://127.0.0.1:5000/api/shanghai/lines?hour=8`
- `http://127.0.0.1:5000/api/shanghai/load-trend?line=all`

## 4. 第三步：启动 Vue 前端

进入前端目录：

```bash
cd frontend
npm install
npm run dev
```

默认地址：

- `http://127.0.0.1:5173`

当前 Vite 已经代理 `/api` 到 Flask 后端，所以只要后端在运行，前端就能直接联调。

## 5. 第四步：使用 demo.html 快速联调

如果你只想快速验证接口和页面，而不想先开 Vue 工程，可以直接用：

- 根目录下的 `demo.html`

使用方式：

1. 先启动 Flask 后端
2. 直接打开 `demo.html`

如果浏览器限制本地文件访问，建议在项目根目录执行：

```bash
python -m http.server 8080
```

然后访问：

- `http://127.0.0.1:8080/demo.html`

## 6. 推荐启动顺序

建议每次都按这个顺序：

1. `python scripts/clean_data.py`
2. `cd backend && python app.py`
3. 打开 `demo.html` 检查接口是否正常
4. `cd frontend && npm run dev`

## 7. 常见问题

### 7.1 页面没数据

通常是还没先运行：

```bash
python scripts/clean_data.py
```

### 7.2 前端打不开图

优先检查：

- Flask 是否已经启动
- `http://127.0.0.1:5000/api/overview` 是否能返回 JSON
- `data/processed/` 是否已经生成

### 7.3 上海页线路图不显示

优先检查：

- `data/processed/shanghai_line_topology.json` 是否存在
- 是否重新执行过 `python scripts/clean_data.py`

## 8. 当前推荐模式

当前开发阶段推荐只使用：

- `DATA_SOURCE=csv`

也就是默认模式，不需要额外设置环境变量。后端会直接读取 `data/processed/` 下的处理后文件。
